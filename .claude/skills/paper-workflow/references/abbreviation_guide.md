# Abbreviation Management

## Rules
- Define every abbreviation at first use in both the abstract and the main text.
- Limit non-standard abbreviations to ≤5 per paper.
- Too many abbreviations disrupt reading flow. Consider whether the abbreviation is truly needed.

## Example
- First use: "physics-informed neural networks (PINNs)"
- Second+ use: "PINNs"
- Do: "PDE" for "Partial Differential Equation" (widely known)
- Don't: "PDA" for "parameter-dependent adaptation" (not standard, unlikely to save space)
